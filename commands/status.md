description: プロジェクトの現状をダッシュボード形式で表示する（読み取り専用）。アプリ名 / MVP 進捗 / 仕様書数 / 手動タスク / 最終テスト / セキュリティ点検履歴を一覧する。
---

このコマンドは **読み取り専用** で、プロジェクトの現状を集計して表示します。何も書き換えません。

## Step 1: アプリの基本情報を集める

`docs/vision/vision.md` を `Read` で読む:
- アプリ名（タイトル行）
- 「ひとことで」セクションの内容
- 作成日 / 最終更新日（`git log -1 -- docs/vision/vision.md --format=%ci` で取得）

ファイルが存在しない場合は「未設定（vision がまだ作成されていません）」と表示。

## Step 2: MVP 進捗を集計

`docs/vision/roadmap.md` を `Read` で読み、`## MVP` セクション内のチェックボックス行を集計:

- `- [x]` の件数 = 完了
- `- [ ]` の件数 = 未着手
- `- [-]` の件数 = スキップ
- 進捗率 = 完了 / 全件 × 100

ファイルが無ければ「未設定（roadmap がまだ作成されていません）」。

## Step 3: 仕様書一覧

`docs/specs/*.md` を `Glob` で列挙（`_template.md` を除外）。各ファイルを `Read` して以下を抽出:
- **ファイル名**（= feature-name）
- 末尾の「ステータス」行（確定 / 更新 / back-fill / ドラフト）
- 最終更新日（`git log -1 -- docs/specs/<file> --format=%ci`）
- **概要**: `## 概要` セクションの最初の非コメント非空行（HTML コメントは除外）。約 40 文字でトランケート。記入が無ければ `(概要未記入)`

## Step 4: 手動タスクを集計

`docs/manual-tasks/*.md` を `Glob` で列挙（`_template.md` 除外）。各ファイルの中身を `Read` で読み:
- `☐ 未着手` または `☐ 着手中` の件数を集計（未完了）
- `☑ 完了` の件数（完了済み、参考）
- 未完了タスクが属する feature-name を保持

## Step 5: 最終アクティビティ

`Bash` で git log を確認:
- 最終コミット: `git log -1 --pretty=format:"%h %ar - %s"`
- 直近のテスト関連コミット: `git log -10 --grep='test\|テスト\|spec' --pretty=format:"%ar - %s" | head -3`

## Step 6: セキュリティ点検履歴

`Bash` で `git log -20 --grep='security\|セキュリティ' --pretty=format:"%ar - %s"` を実行。

- ヒットあり → 直近のものを表示
- ヒット無し かつ `docs/specs/` に `auth|login|payment|user-info|profile|signup|password` を含むファイル名があれば、「セキュリティ感度の高い機能がありますが security-review が見当たりません。`/cc-development-team:security-review` の実行を推奨」と警告

## Step 7: 整形して表示

以下のフォーマットでユーザーに出力:

```
【プロジェクト現状ダッシュボード】

# 基本情報
- アプリ名: <名前>
- 概要: <ひとこと>
- vision 最終更新: <日付>

# MVP 進捗
☑ 完了: N 件 / 未着手: M 件 / スキップ: K 件
進捗率: XX%

完了済み: <feature 名のリスト>
残り: <feature 名のリスト>

# 仕様書 (N 件)
- <feature-name>.md (ステータス, 最終更新) — <概要>
- ...

# 手動タスク
- 未完了: N 件
  - <タスク名> in <feature-name>
  - ...
(全て完了の場合は「全て完了」と表示)

# 最終アクティビティ
- 最終コミット: <時刻 / メッセージ>
- 直近のテスト関連コミット: <あれば 3 件まで>

# セキュリティ点検
- 履歴: <あり / 見当たらず>
- 推奨アクション: <あれば>

# 次のオススメ
<文脈に応じた 1〜2 個の提案>
```

## 「次のオススメ」の出し方

状態に応じて以下の優先順で 1〜2 個を提案:

1. **手動タスクが未完了** → 「手動タスクを完了させてください」と該当を提示
2. **MVP に未着手機能あり** → 「次は `<feature-name>` を `/cc-development-team:design <feature-name>` で設計しましょう」
3. **認証/決済/PII 機能ありで security-review 未実施** → 「`/cc-development-team:security-review` の実行を推奨」
4. **MVP 全完了 + 手動タスク完了** → 「`/cc-development-team:release-check` でリリース準備状況を確認しましょう」
5. **何も未完了が無い場合** → 「現状特に対応すべきことはありません。お疲れさまです」

## やってはいけないこと

- **ファイルを書き換える**（このコマンドは読み取り専用）
- ファイルが無いのに推測でデータを捏造する（「未設定」「不明」と素直に書く）
- 古い情報を「現在」として表示する（git log で日付を確認すること）
